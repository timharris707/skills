# v1.13 fix-it — design-brief inputs (fact sheet)

_Scouted 2026-07-02 against `main` @ `dfa4882` (post-v1.12.0) as input for the v1.13 Phase 1 dogfood design roundtable. Paths relative to `skills/advisory-board/`. This is a working design document, not user-facing docs._

## 1. Synthesizer spawn path (what P2's revision seat generalizes)

- Template + sha discipline: `SYNTHESIZER_TEMPLATE` (`_conductor/synthesizer.py:90-179`), versioned `advisory-board/synthesizer@2` (:185), sha256 via `synthesizer_template_sha()` (:188), recorded in `run-recipe.yaml` (`recipe.py:251`) and printed at spawn (`cli.py:515`).
- DATA-fence: `SYNTHESIZER_BEGIN/END_MARKER` (:68-69); `neutralize_synth_markers()` (:72-79) strips marker echoes before splicing.
- Seat choice: `choose_synthesizer_seat()` (:205-233) — `--synthesizer-seat` (must be a board seat: "egresses to a provider already covered by the run's disclosure"), else `claude` if seated, else first usable seat, else `config.board[0]`. Neutrality comes from the PROMPT (no lens, no source), not the seat name.
- Skeleton/merge: `build_skeleton()` (:236-298); `PROTECTED_SKELETON_KEYS` (:195) and `LIFECYCLE_KEYS` (:202) kept separate (prompt enumeration stays byte-stable); `merge_synthesizer_content()` (:459-485) strips both, re-pins `lens_preset`, computes `unanimous` itself.
- JSON extraction: `extract_json_object()` (:386-419) — fenced block preferred, brace-balanced scan fallback, LAST candidate wins.
- Validation: `validate_verdict()` (:488-517) → `board_verdict.validate`, SystemExit captured to a plain error string.
- Failure posture: refuses synthesis if any non-dropped seat lacks a VERDICT token (:564-588, no spawn). Two attempts, retry only on `RETRYABLE_FAILURES` = timeout|invalid (`spawn.py:172`). Shape classifier `_classify_synthesizer_shape()` (:663-696). CLI (`cli.py:497-613`): success → `verdict.json`; failure → `verdict-rejected.json` + loud warning + **exit 0 by default** ("a synth hiccup never discards the successful rounds"); `--strict-exit` → `EXIT_NO_VERDICT` (4).
- Black-box record: `render_synthesizer_raw()` (:711-746) → `synthesizer/<seat>.raw`.

**Reusable as-is:** `spawn()` + retry set, fence/neutralize pattern, `extract_json_object`, `PacketBlob`/packet-hash egress plumbing, raw-recorder pattern, protected-key merge pattern, seat-resolution shape.
**Needs generalizing:** shape classifier (revision reply is revised text + changes, not JSON-only — maybe two-part reply or two spawns), skeleton/merge (verdict-schema-specific), validator (changes needs its own), the template itself.

## 2. `--output` today is DECORATIVE — no dispatch exists

- `render_verdict.py --shape` (:917-919) actually selects rendering (`full-handoff|quick-verdict|implementation-sequence`).
- `run --output` (`cli.py:705-706`, same three choices) is only printed (run-card `artifacts.py:103`, metadata `:373`) and persisted (`recipe.py:240`). **No code branches on `config.output`.** P2's `--output revised-draft` must BUILD the dispatch (natural shape: fourth enum + `_run_revision_step()` after synthesis succeeds, analogous to `_run_synthesis_step()`), not extend an existing switch.

## 3. Finding identity — NO stable ids exist

- Schema items are `{title, body, evidence[]}` — no `id` anywhere (`references/verdict-schema.md:30-46`; validator never checks one).
- De facto join key = exact `title` (precedent: `amend --on` via `_finding_titles()`, `board_verdict.py:525-532`, strict match or die listing titles).
- `delta.py` matching is tiered mechanical (exact normalized title → shared concrete citation → SequenceMatcher ≥0.75 + shared token), explicitly not id-based; its normalization is reusable if titles-as-keys is chosen.
- **Roundtable question:** introduce a real `id` on blockers/concerns (schema change → needs a D-decision + compat test per the D8 pattern) vs key `changes` by title (existing precedent; inherits title-rename fragility).

## 4. No diff machinery exists in the render engine

- `_render_engine.py` is pure token/block substitution — zero diff/ins-del support.
- v1.12 delta section renders plain bulleted title lists (`render_delta_markdown`, `render_verdict.py:283-309`).
- Only difflib in repo: `unified_diff()` for prompt text (`revise.py:build_source_diff`, :249-275, capped `DIFF_MAX_LINES`) and `SequenceMatcher.ratio()` (delta). **`get_opcodes()` unused anywhere** — P3's ins/del spans are genuinely new work.

## 5. No source-type (prose vs code) signal exists

- `SourceSpec.kind` = transport (`path|url|stdin`), not content type (`config.py:51-58`, `:151-179`); recipe persists the same.
- `lens_preset` is a human-chosen review framing, not a content discriminator, and never drives output format.
- **Roundtable question:** extension heuristic vs explicit `--source-type prose|code` flag vs lens inference (imprecise).

## 6. Artifacts + egress posture

- Artifact tree authority: `render_artifact_tree()` (`artifacts.py:206-239`). Natural v1.13 landing: `revision/<seat>.md`+`.raw` (mirroring `synthesizer/`) + top-level `revised-draft.md` / `revised-draft.patch` + `changes.json`, gated `if config.output == "revised-draft"` analogous to `if config.synthesize`.
- **No new egress:** every seat's round-1 prompt already embeds the full source (`egress.py:build_packet`, :95-111); the verdict derives from the seats' own words. A revision seat on a board seat's adapter sees nothing its provider hasn't seen — disclosure needs only a purpose mention, no new category.

## 7. The reserved `changes` key — exact language

- Validator refusal (`board_verdict.py:137-144`): `"changes is reserved for the revision artifact (v1.13) and not yet defined; remove it"`. `LIFECYCLE_FIELDS`/`LIFECYCLE_KEYS` include it; synthesizer merge strips it.
- Schema doc (`verdict-schema.md:138-140`): "`changes` — **reserved** for the v1.13 revision artifact (the edit → finding mapping of `--output revised-draft`)."
- **Ambiguity for the roundtable:** the reservation is a top-level key ON `verdict.json`, but the roadmap names a separate FILE `changes.json`. Decide: (a) separate artifact file only (reservation was name-squat prevention), or (b) `verdict.json` gains a `changes` key (pointer/summary or the mapping itself), consuming the reservation.

## 8. Roadmap anchors

- Decisions format: `## Decisions`, entries `- **D<N>** <settled past-tense paragraph>` (match D8's style). Existing **D6**: "Transform never touches the source — `--output revised-draft` writes new artifacts only; applying the revision is the human's act." Risk **R4**: fix-it must not read as the board rewriting your work (artifact-only, opt-in, per-seat endorsement, human applies).
- v1.13 phases: P1 design roundtable (decisions: redline format per source type, changes.json shape, endorsement-pass shape/default, failure posture on conflicting findings) → P2 revision seat + changes.json (#2) → P3 difflib-opcode redlines + fenced citation snippets (#12) → P4 endorsement pass + release.

## Flags the roundtable must address (roadmap glosses these)

1. `--output revised-draft` has no dispatch path — new plumbing, not a new enum value.
2. "keyed to blocker/concern ids" presupposes ids that don't exist — invent an id or key by title.
3. `changes` key (on verdict.json) vs `changes.json` (separate file) — the reservation and the roadmap name different things.
4. No prose-vs-code detection exists to drive redline-vs-patch.
5. No ins/del rendering exists — P3 is new machinery, not wiring.
